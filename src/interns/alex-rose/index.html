Now, to the fun stuff:

In my opinion, software engineering can be generalized to into 4 different steps:

  1. “I want to build something”
  2. “I need to solve these problems to build the thing”
  3. "I have solved the problems but now want to improve the thing"
  4. “I need to solve these problems to improve the thing”

This cycle goes on and on until you either ditch the project or start something new. Luckily, I love building things and solving problems.

There’s a misconception about coding, that it’s all about math and algorithms. To an extent, this is true, but it misses the best thing about coding, which is the creativity of it at all. Each problem can be solved in very different ways and finding your own solution (at least to more complex problems) usually requires some creativity. This is a long way of saying that I want to be a software engineer because I love creating things.
